Regular Function  যখন Function Declaration , তখন তা Named Function । Regular Function  যখন Function Expression , তখন তা Named Function  বা Anonymous Function  দুইটাই হতে পারে। অন্যদিকে, Arrow Function  সদা Anonymous ।

Regular Function-এর তুলনায় Arrow Function-এর সিনট্যাক্সে বাঁচালতা কম (মানে কী-ওয়ার্ড বা টোকেনের ব্যবহার কম)। যেমন— function  কীওয়ার্ড লিখতে হয় না। প্যারামিটার একটা হলে প্যারেন্থিসিস দরকার হয় না। আবার কখনো কখনো (কখন? সবাই জানে!) যে ফাংশন বডির ব্র্যাসেস ও বডির ভেতরে return  কীওয়ার্ড বাদেই ভ্যালু রিটার্ন করা যায়, তাতে কে মুগ্ধ হয় না?

Regular Function  প্রতিটি ইনভোকেশনে ইনভোকেশনের ধরণ অনুযায়ী নিজের ভিতরে this  কীওয়ার্ডের নিজস্ব মান তৈরি করে। অন্যদিকে, Arrow Function  তা করে না— সে সবসময় প্যারেন্ট লেক্সিক্যাল কনটেক্সটের this-এর মান ধরে রাখে।

নিজস্ব this  তৈরি না করার কারণে Arrow Function  কখনো Object Constructor  হিসেবে ব্যবহার করা যায় না।

Regular Function-এর সাথে আরেকটা বৈসাদৃশ্য— Arrow Function-এর ভেতরে arguments  বা super-এর নিজস্ব বাইন্ডিং নেই।

Event Loop হলো Call Stack-এর অতন্দ্র প্রহরী। Event Loop সদা Call Stack-এর দিকে নজর রাখে।

Event Loop যখনই Call Stack-কে খালি পায়, তখনই সে Render Queue ও Message/Event Queue-এর দিকে তাকায়, এবং এই Queue দুইটির যেকোনো একটিতে (যথাক্রম প্রাধান্যে) কোনো Task পেলে, Queue-এর শুরু থেকে একটি Task নিয়ে সেটিকে Call Stack-এ Execute করে।

জাভাস্ক্রিপ্টে এক্সপ্রেশন ভ্যালু হিসেবে বিভিন্নভাবে undefined আসতে পারে। যেমন—

  1. var বা let দিয়ে ডিক্লেয়ার করা ভ্যারিয়েবলের ভ্যালু ডিক্লারেশন স্টেট্মেন্টে undefined-এ ইনিশিয়ালাইজ করলে বা undefined বাদে অন্যকিছুতে ইনিশিয়ালাইজ না করলে, ভ্যারিয়েবলের মান undefined থাকে।
  2. var দিয়ে ডিক্লেয়ার করা ভ্যারিয়েবলের ডিক্লারেশন স্টেট্মেন্টের আগে ভ্যারিয়েবলটির ফাংশন স্কোপে (বা গ্লোবাল কনটেক্সটে গ্লোবাল স্কোপে) ভ্যারিয়েবলের মান undefined থাকে।
  3. ফাংশন undefined রিটার্ন করলে বা ফাংশন এক্সপ্লিসিটলি কিছু রিটার্ন না করলে, ফাংশন ইনভোকেশন এক্সপ্রেশনের মান undefined হয়।
  4. ফাংশন ইনভোকেশনের সময় যেসকল প্যারামিটারের বিপরীতে আর্গুমেন্ট দেওয়া হয় না, ফাংশন বডির ভেতরে সেসকল প্যারামিটারের মান undefined হয়, যদি না ওই প্যারামিটারের ডিফল্ট ভ্যালু থাকে।
